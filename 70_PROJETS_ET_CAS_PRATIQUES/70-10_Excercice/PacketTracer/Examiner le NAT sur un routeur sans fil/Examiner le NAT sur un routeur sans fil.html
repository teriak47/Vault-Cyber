Incomplet Packet Tracer – Examiner le NAT sur un routeur sans fil

Objectifs

  • Examiner la configuration NAT sur un routeur sans fil
  • Installer 4 ordinateurs pour se connecter à un routeur sans fil via DHCP
  • Analyser le trafic réseau à l'aide de la fonction NAT

Instructions

Partie 1: Examinez la configuration permettant d'accéder au réseau externe.

1.  Ajoutez 1 PC et connectez-le au routeur sans fil avec un câble droit. Avant de passer à l'étape suivante, attendez que tous les voyants de liaison passent au vert ou cliquez sur Fast Forward (Avance rapide).

2.  Sur le PC, cliquez sur Desktop (Bureau). Sélectionnez Configuration IP. Cliquez sur DHCP pour permettre à chaque appareil de recevoir une adresse IP via DHCP sur le routeur sans fil.

3.  Notez l'adresse IP de la passerelle par défaut. Fermez la Configuration IP lorsque vous avez terminé.

4.  Naviguez vers le navigateur Web et entrez l'adresse IP de la passerelle par défaut dans le champ URL. Saisissez le nom d'utilisateur admin et le mot de passe admin lorsque vous y êtes invité.

5.  Cliquez sur l'option de menu Status (Statut) dans le coin supérieur droit. Ceci ouvre la page de sous-menu du routeur.

6.  Faites-la défiler vers le bas jusqu'à l'option Connexion Internet. L'adresse IP attribuée ici est l'adresse attribuée par le Fournisseur d'accès à Internet (FAI). Si aucune adresse IP ne s'affiche (0.0.0.0 apparaît), fermez la fenêtre, patientez quelques secondes et réessayez. Le routeur sans fil est en train d'obtenir une adresse IP auprès du serveur DHCP du FAI.

L'adresse affichée ici correspond à l'adresse attribuée au port Internet du routeur sans fil.

Question :

S'agit-il d'une adresse privée ou publique?

Zone de réponse

C'est une adresse IP publique

Partie 2: Examinez les paramètres permettant d'accéder au réseau interne.

1.  Cliquez sur Local Network (Réseau local) dans la barre de sous-menu de la fenêtre État.

2.  Faites défiler la page vers le bas jusqu'aux informations sur le réseau local. Il s'agit de l'adresse attribuée au réseau interne.

3.  Faites défiler encore la page vers le bas jusqu'aux informations sur le serveur DHCP et la plage d'adresses IP qui peuvent être attribuées aux hôtes connectés.

Question :

S'agit-il d'adresses privées ou publiques?

Zone de réponse

Ce sont des adresses IP privées

4.  Fermez la fenêtre de configuration du routeur sans fil.

Partie 3: Connectez 3 ordinateurs au routeur sans fil.

1.  Ajoutez 3 autres PC et connectez-les au routeur sans fil avec des câbles droits. Avant de passer à l'étape suivante, attendez que tous les voyants de liaison passent au vert ou cliquez sur Fast Forward (Avance rapide).

2.  Sur chaque ordinateur, cliquez sur Desktop (Bureau). Sélectionnez Configuration IP. Cliquez sur DHCP pour permettre à chaque appareil de recevoir une adresse IP via DHCP sur le routeur sans fil. Fermez la Configuration IP lorsque vous avez terminé.

3.  Cliquez sur Command Prompt (Invite de commande) pour vérifier la configuration IP de chaque périphérique à l'aide de la commande ipconfig /all.

Remarque: Ces périphériques recevront une adresse privée. Les adresses privées ne pouvant pas parcourir Internet, une traduction NAT est requise.

Partie 4: Affichez la traduction NAT sur le routeur sans fil.

a.  Passez en mode Simulation en cliquant sur l'onglet Simulation, situé dans l'angle inférieur droit. L'onglet Simulation est situé à côté de l'onglet Temps réel et comporte un symbole de chronomètre.

b.  Visualisez le trafic en créant un PDU complexe en mode Simulation:

·  Dans le panneau de simulation, cliquez sur Show All/None (Afficher tout/aucun) pour supprimer tous les événements visibles. Cliquez ensuite sur Edit Filters (Modifier les filtres), puis, sous l'onglet Misc (Divers), cochez les cases TCP et HTTP. Fermez la fenêtre lorsque vous avez terminé.

·  Ajoutez un PDU complexe en cliquant sur l'enveloppe ouverte située dans le menu supérieur.

·  Cliquez sur l'un des PC pour le spécifier comme source.

c.  Spécifiez les paramètres de la PDU complexe en modifiant les éléments suivants dans la fenêtre:

·  Sous PDU Settings (Paramètres PDU) > Sélectionnez l'application doit être réglé sur: HTTP.

·  Cliquez sur le serveur ciscolearn.nat.com pour le spécifier comme périphérique de destination. 

·  Pour le Port source, entrez 1000.

·  Dans la section Paramètres de simulation, sélectionnez Periodic (Périodique). Saisissez 120 secondes dans le champ Intervalle.

·  Cliquez Create PDU (Créer un PDU) dans la fenêtre Créer un PDU complexe.

d.  Double-cliquez sur le volet de simulation pour le déverrouiller à partir de la fenêtre PT. Vous pouvez ainsi déplacer le volet de simulation et afficher l'ensemble de la topologie du réseau.

e.  Observez le flux de trafic en cliquant sur Play (Lecture) dans le panneau de simulation. Accélérez l'animation en déplaçant le curseur de lecture vers la droite.

Remarque: Cliquez sur View Previous Events (Afficher les événements précédents) quand le message entier du buffer s'affiche.

Partie 5: Affichez les informations d'en-tête des paquets ayant circulé sur le réseau.

a.  Analysez les en-têtes des paquets envoyés entre l'ordinateur et le serveur web.

1.  Dans le Simulation Panel (volet de simulation), cliquez deux fois sur la troisième ligne de la liste des événements. Une enveloppe représentant cette ligne s'affiche alors dans la zone de travail.

2.  Cliquez sur l'enveloppe dans la fenêtre de la zone de travail pour afficher le paquet et les informations d'en-tête.

b.  Cliquez sur l'onglet Entrée de l'unité de données de protocole. Observez les informations sur le paquet pour l'adresse IP source (SRC) et de destination.

c.  Cliquez sur l'onglet Sortie de l'unité de données de protocole. Observez les informations sur le paquet pour l'adresse IP source (SRC) et de destination.

Remarquez le changement de l'adresse IP SRC.

d.  Cliquez sur d'autres lignes d'événements pour afficher ces en-têtes lors de la procédure.

e.  Lorsque vous avez terminé, cliquez sur Vérifier les résultats pour vérifier votre travail.
